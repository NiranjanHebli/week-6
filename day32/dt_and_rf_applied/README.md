# Day 32 - PM -  Decision Trees & Random Forest : Applied

## Part A - Concept Application

This task involves building a insurance fraud prediction pipeline using scikit-learn to compare Decision Tree and Random Forest on synthetic claims data. Included rule extraction, recall-based RF tuning, metrics comparison, and cost-sensitive evaluation where false negatives were weighted 10x higher than false positives.

## Key Learnings

- Recall matters most in fraud detection because missed fraud is costly.
- Decision Trees help with explainability, while Random Forest improves performance.
- Cost-sensitive evaluation helps align model choice with business impact.
